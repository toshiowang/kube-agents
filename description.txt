# PR Title: feat: Introduce DevTeam Agent, Cooperative Routing Playbooks, and Installer Upgrades

## Overview

This Pull Request delivers the **initial release of the DevTeam Agent (`devteam`)** and introduces a **Dual-Agent Cooperative Layout** to `kube-agents`. 

Through **Phase 6** of the installer, this PR automatically and idempotently configures the "main" conversational agent's playbooks (`AGENTS.md`, `TOOLS.md`, and `ROUTING.md`) to enable seamless, visible, and verified task delegation. It also expands the documentation with complete Prerequisites and Getting Started guides, completing the foundational harness for cooperative DevOps agents.

---

## Key Achievements

### 1. DevTeam Agent Integration & Pre-Bundled Skills
*   **Production-Safety Coach Persona**: Created the `devteam` agent (`IDENTITY.md` and `SOUL.md`) branded as the *Production-Safety Coach & Workload Custodian*. It represents developers' interests, focusing on feature velocity, application safety, and strict SLO protection (Spot VM safety thresholds) without requiring direct developer cluster access.
*   **10 Core Responsibilities**: Populated detailed guidelines for Interface Management (troubleshooting in Slack), PR Review (enforcing CPU/memory limit requests and Pod Security Standards), Workload Optimization, Automated RCA, and Supply Chain SBOM/CVE security.
*   **7 Scheduled Cron Tasks**: Configured `HEARTBEAT.md` with 7 automated tasks (rollout watch, error rate log monitors, service health checks, hourly SLO compliance monitors, daily cost audits, daily latency baselines checks, and weekly Platform compliance scans) utilizing `memory/heartbeat-state.json` for dynamic, token-efficient schedule rotation.
*   **9 Pre-Bundled Domain Skills**: Pre-copied 9 specialized skills (onboarding, observability, productionize, reliability, scaling, hardening, cost-optimization, cost-analysis, and AI model onboarding) directly into the agent's folder (`devteam/skills/`) for instant out-of-the-box capability.

### 2. Installer Phase 6: Main Agent Playbook Provisioning
Upgraded `openclaw/scripts/install.sh` to append Phase 6, which dynamically and **idempotently** configures the primary main agent's workspace files at `~/.openclaw/workspace/`:
*   **`AGENTS.md`**: Appends the **Agent Delegation Policy** (ownership matrix, main agent responsibilities, strict Git/image/rollout proof gates, and fallback policies).
    *   *Relay Bullet*: Directs the main agent to automatically *Relay cross-agent chat mentions so requests/responses remain visible*.
*   **`TOOLS.md`**: Appends **Agent Routing Shortcuts** (`@devteam`, `@operator`, `@main`), strict subagent delivery checklists, and development safety rails.
*   **`ROUTING.md` Playbook**: Creates the complete, practical handoff playbook.
    *   *Cross-Agent Relay Rule*: Establishes the strict rule for **chat-visible coordination**. If a subagent mentions another subagent (e.g. `@devteam ...` inside an operator chat in TUI/Telegram), `main` **MUST** relay the instruction and mirror the response back into the same thread so the human operator has complete visibility over cross-agent handoffs.

### 3. Documentation Refinements
*   **Top-Level README (`README.md`)**: Added a critical **Prerequisites** section (detailing OpenClaw version `2026.5.3`+, `curl`, `jq`, and cluster context credentials) to prevent user installation issues.
*   **OpenClaw README (`openclaw/README.md`)**: Completely restructured to document the cooperative dual-agent layout, routing commands, and responsibility matrix. Expanded the **Getting Started** section to explain:
    *   *Main Coordinator chat* (`openclaw tui`): Chatting with `main` to test dynamic delegation and routing commands.
    *   *Direct Subagent chat*: How to connect directly to `operator` or `devteam` sessions using their respective session keys.

## Verification Results

### Successful Local Installation Test
The entire dual-agent package was successfully verified via local/offline mode on an Ubuntu server:
```bash
LOCAL_DIR="/path/to/kube-agents" bash /path/to/kube-agents/openclaw/scripts/install.sh
```

The installation ran flawlessly:
- **Dynamic Discovery**: Auto-detected, registered, and configured both `operator` and `devteam` agents.
- **State Playbooks**: Successfully and idempotently appended playbooks to `AGENTS.md` and `TOOLS.md`, and created `ROUTING.md` with correct cross-agent relay rules.
- **Idempotent Re-runs**: Verified that running the installer multiple times is 100% safe and does **not** duplicate configuration text blocks.
